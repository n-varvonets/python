Lamda - сердце Serverless технологий:
    - Сервис позволяюзий запускать програмньій код без ваших таковьіх работающих серверов.
        напоминает "храниемую процедуру" for aws-сервисов
    - Ваш програманьій код запускается и бежит на компьютерах "єластичной можности",
        "єластичной можности" - сервис сам делает  AutoScaling в зависимости от ганрузки

например:
    1)Если файл отправляется в S3 bucket, то что-то сделать с ним (разахивировать, конвертировать, переименовать и т.д.)
    2) когда кто-то переходит по url нашего инстанса или отдельной какой-то страницьі
    2) изменения в RDS or DynamoDB(NoSql)

Запускаетс через:
    - aws ui
    - asw cli
    - API Gateway: позволить создать url-link, перейдя по которой GET - можно ее вьізвать
    - sns, sqs

Practice: создадить 2 Lambda funcs, которьіе:
    - одна читает существуеющие S3 buckets  у меня
    - вторая: создавать s3 buckets

Use Lambda Proxy integration - позволит использовать ламбду, как url, т.е. методом get передавать параметрьі в events
