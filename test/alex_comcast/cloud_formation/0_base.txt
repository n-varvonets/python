Что такое AWS CloudFormation?
    AWS CloudFormation позволяет создать свой шаблон в формате JSON или YAML, который будет представлять
    собой простой текстовый файл, описывающий все ресурсы AWS для развертывания и запуска вашего приложения.

Как работает AWS CloudFormation?
    - Вы кодируете свою инфраструктуру в текстовом файле YAML или JSON, который называется шаблоном.
    - Создайте свой код локально в инструменте AWS CloudFormation или загрузите файл YAML или JSON в корзину S3.
    - Теперь вы можете использовать графический интерфейс AWS CF или интерфейс командной строки для создания стека на основе кода вашего шаблона.
    - Наконец, CloudFormation развернет ресурсы, предоставит их и настроит указанный вами шаблон.

AWS CloudFormation vs Elastic Beanstalk?
     Elastic Beanstalk используется для создания среды любой платформы программирования
     и легкого развертывания ваших приложений и запуска их в облаке.

Преимущества AWS CloudFormation

    - Автоматизация инфраструктуры
    - Контроль безопасности
    - Расширяемость

Стеки CloudFormation :
    Стек — это набор различных ресурсов AWS, объединенных в единое целое.
    Вы можете создавать, обновлять или удалять наборы ресурсов AWS, создавая, обновляя и удаляя стеки.

    Например, стек может содержать все ресурсы, необходимые для запуска веб-приложения,
    такого как веб-сервер, база данных и сетевые правила. Вы можете просто обновить стек,
    и все веб-приложение будет обновлено. Если вам больше не нужно приложение,
    вы можете просто удалить стек, и все ресурсы будут удалены.

    вопросы к интервью AWS :
         - AWS CloudFormation позволяет создавать или удалять ресурсы стека только по мере необходимости.
          Поскольку ресурсы рассматриваются как единое целое, все они должны быть успешно созданы или удалены.
          - Если ресурс не может быть создан, весь стек отбрасывается или удаляется.
          - Если ресурс нельзя удалить, другие ресурсы сохраняются до тех пор, пока стек не будет полностью удален.
          Вы можете работать со стеками, используя консоль AWS CloudFormation, API или интерфейс командной строки AWS.

Шаблон CloudFormation :
    Ресурсы, которые вы хотите предоставить в своих стеках CloudFormation, описаны в этих шаблонах.

    В шаблоне есть несколько разделов:

        обьязательно должно бьіть описано:
            - Ресурсы: Указывает ресурсы стека и их свойства, например экземпляр
              Amazon Elastic Compute Cloud или корзину Amazon Simple Storage Service.

        все остальное - опицонально:
            - Версия формата: определяет возможности шаблона.
            - Описание: Текстовая строка, описывающая шаблон
            - Метаданные: Дополнительная информация о шаблоне
            - Параметры: Значения для передачи в ваш шаблон во время выполнения
            - Сопоставления: сопоставление ключей и связанных значений, которые вы можете использовать для указания значений условных параметров, аналогично таблице поиска
            - Условия: Дополнительные условия создания ресурсов
            - Преобразование: Необязательный раздел Transform указывает один или несколько макросов, которые AWS CloudFormation использует для обработки вашего шаблона.
            - Выходы: Описывает значения, которые возвращаются всякий раз, когда вы просматриваете свойства вашего стека.
