1) у тебя должен быть cloudFormation стек
2) в этом стеке должно быть все, что необоходимо для работы твоего приложения
3) 4 ендпоинта GET POST PUT DELETE которые работают с  Item (элемент в БД)  по ID-шнику. Придерживаться атомарности, т.е. одна лямба функция = один метод.
4) хранение Items в DynamoDB. структура:
    - primary key - ID.
    - тело ITEM ( которое приходить в payload) храниться как JSON
    - 2 поля добавляются автоматически (added, updated) при создании и обновлении ITEM в DB
5) дополнительный эндпоинт, который принимает ID, если такого элемента нет - 404 ошибка, если есть отправляется тело ITEM на SNS топик, а на топик в свою очередь есть подписка (Subscription) на отправку имейла с телом ITEM на почту
6) желательно увидеть какие то логи во всех логгруппах (инфа об создании, удалении, обновлении, какой тип запроса поступил, время запроса)
7)при желании можно добавить ещё 1 GET ендпоинт без передачи ID, он возвращает все элементы. Оставить ифнормацию об  самих елементах и их кол-ве, а лишнее убрать.
8)при желании можно добавить DNS в стек для роута апи
